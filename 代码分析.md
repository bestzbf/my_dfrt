# D4RT 代码分析文档

## 项目概述

D4RT (Dynamic 4D Reconstruction and Tracking) 是一个用于动态4D重建和跟踪的深度学习模型，基于Transformer架构实现视频中的点跟踪、深度估计和相机姿态估计。

## 核心架构

### 模型结构 (models/d4rt.py)

**D4RT模型**采用编码器-解码器架构：

- **编码器 (Encoder)**: 将视频编码为全局场景表示
  - 支持多种变体: base, large, huge, giant
  - 可选VideoMAE预训练初始化
  - 输入: 视频张量 (B, T, C, H, W)

- **解码器 (Decoder)**: 解码时空查询预测3D位置
  - 查询接口: (u, v, t_src, t_tgt, t_cam)
  - 支持任务: 点跟踪、点云生成、深度图、相机参数估计

### 数据处理 (data/dataset.py)

**PointOdysseyDataset** 处理PointOdyssey数据集：

核心功能：
- 快速注释缓存 (`anno_fast/`)
- 打包帧缓存 (RGB/Depth/Normal的.bin文件)
- 运动边界缓存 (用于采样策略)
- 支持数据增强和多种采样策略

关键文件结构：
```
sequence/
├── anno_fast/
│   ├── trajs_2d.npy          # 2D轨迹
│   ├── trajs_3d.npy          # 3D轨迹
│   ├── valids.npy            # 有效性掩码
│   ├── intrinsics.npy        # 相机内参
│   ├── extrinsics.npy        # 相机外参
│   ├── rgb_frames.bin        # 打包的RGB帧
│   ├── depth_frames.bin      # 打包的深度帧
│   └── motion_boundary_*.npy # 运动边界缓存
```

### 损失函数 (losses/losses.py)

**D4RTLoss** 包含多个损失项：
- `loss_3d`: 3D位置损失
- `loss_2d`: 2D投影损失
- `loss_vis`: 可见性损失
- `loss_disp`: 视差损失
- `loss_normal`: 法线损失
- `loss_conf`: 置信度损失

## 数据准备脚本

### prepare_pointodyssey_local_dataset.py

**功能**: 将PointOdyssey数据集转换为SSD友好的快速加载格式

**主要步骤**:
1. 复制或构建快速注释缓存 (NPY格式)
2. 构建打包帧缓存 (BIN + offsets)
3. 生成运动边界缓存 (多个stride)
4. 可选复制原始文件

**使用示例**:
```bash
python scripts/prepare_pointodyssey_local_dataset.py \
  --src-root /data2/datasets/PointOdyssey \
  --dst-root /data2/datasets/PointOdyssey_fast \
  --splits train val test \
  --workers 8 \
  --verify-sample
```

**优化特性**:
- 并行处理多个序列
- 增量构建（跳过已存在的缓存）
- 仅保留运行时必需文件，减少存储
- 支持验证生成的数据集

## 关键优化

### 1. 快速数据加载
- 预计算注释缓存避免重复解析NPZ
- 打包帧缓存减少文件系统开销
- 运动边界预计算加速采样

### 2. 内存效率
- 使用mmap模式读取大型数组
- 打包存储格式 (offsets + binary)
- 可选的原始文件排除

### 3. 训练加速
- 分布式训练支持 (DDP)
- 多worker数据加载
- 预计算局部patch

## 数据流

```
原始PointOdyssey数据
    ↓
prepare_pointodyssey_local_dataset.py
    ↓
快速缓存格式 (anno_fast/)
    ↓
PointOdysseyDataset
    ↓
DataLoader → 训练批次
    ↓
D4RT模型 (Encoder + Decoder)
    ↓
D4RTLoss → 反向传播
```

## 性能特点

- **缓存优先**: 所有运行时数据预计算
- **并行友好**: 多进程数据准备和加载
- **存储优化**: 仅保留必需文件，减少70%+存储
- **灵活采样**: 基于运动边界的智能采样策略
